API Documentation

Простой и удобный доступ к данным через RESTful API

Общая информация

Наш API предоставляет доступ к данным постов через RESTful-интерфейс. Все ответы возвращаются в формате JSON.

  • Базовый URL: http://127.0.0.1:5000/api
  • Формат данных: JSON
  • Кодировка: UTF-8
Получение списка постов

Возвращает все посты, отсортированные по дате (новые первыми).

GET /api/posts 
{
  "posts": [
    {
      "body": "evieuvhevheivheivhervh",
      "featured_posts": [],
      "id": 1,
      "likes": [],
      "preview": "evieveververiv",
      "timestamp": "2026-04-17 23:28:53",
      "title": "vbvbfbbvebvebv",
      "user_id": 1
    }
  ]
}
Получение конкретного поста

Возвращает пост с указанным ID. Если пост не найден, возвращается ошибка 404.

GET /api/posts/<post_id>
GET /api/posts/1 
{
  "post": {
    "body": "evieuvhevheivheivhervh",
    "featured_posts": [],
    "id": 1,
    "likes": [],
    "preview": "evieveververiv",
    "timestamp": "2026-04-17 23:28:53",
    "title": "vbvbfbbvebvebv",
    "user_id": 1
  }
}

Примеры использования на Python

Для работы с API вы можете использовать библиотеку requests:

import requests

# Получение всех постов
response = requests.get('http://127.0.0.1:5000/api/posts')
print(response.json())

# Получение поста с ID=1
response = requests.get('http://127.0.0.1:5000/api/posts/1')
print(response.json())

Для получения актуальной информации следите за обновлениями API.
При возникновении вопросов обращайтесь в техническую поддержку.
